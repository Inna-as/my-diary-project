# Блог-платформа "Мой Дневник"

Учебный проект персонального дневника с древовидной системой комментариев, продвинутым поиском, сортировкой, профилями авторов и кэшированием данных.

## 🛠 Выполненные технические требования
- **Фреймворк**: Python 3.12, Django 5.x.
- **База данных**: SQLite, взаимодействие через Django ORM.
- **Кэширование (п. 2.1)**: Подключен Redis (`django-redis`). Реализовано кэширование фрагментов шаблонов (`{% cache %}`) для боковой панели популярных тегов и механизм автоматической инвалидации кэша при добавлении новых статей. Конфигурация адаптирована под Windows-окружение сервера.
- **Аутентификация (п. 2.2)**: Регистрация, вход и выход пользователей с разграничением прав доступа (изменять и удалять записи может только автор).
- **CRUD-операции (п. 2.2)**: Полный цикл управления статьями и древовидная структура комментариев.
- **Продвинутый поиск и фильтрация (п. 2.2)**: Полнотекстовый поиск по ключевым словам в заголовках, тексте статей и авторам с использованием `Q`-объектов. Фильтрация статей по клику на теги.
- **Интерактивная сортировка (п. 2.2)**: Переключение отображения статей (Сначала новые, Сначала старые, Самые обсуждаемые) с использованием ORM-агрегации `Count`.
- **Профили авторов (п. 2.2)**: Персональные страницы пользователей с выводом их личной ленты постов и агрегированной статистики (общее количество статей и оставленных комментариев).
- **Пагинация (п. 2.2)**: Постраничный вывод (по 3 записи на страницу) с капсульным дизайном и сохранением всех активных фильтров (поиск, теги, сортировка) при переходах.
- **Кастомный Middleware (п. 2.3)**: `SimplePerformanceMiddleware` для замера времени ответа сервера.

## 🚀 Инструкция по развертыванию

1. **Клонируйте репозиторий и создайте виртуальное окружение:**
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Для Linux/macOS
   .venv\Scripts\activate     # Для Windows
   ```
2. **Установите зависимости проекта:**
   ```bash
   pip install -r requirements.txt
   ```
3. **Настройте переменные окружения:**
   В корневой директории проекта создайте файл `.env` и добавьте в него следующие настройки:
   ```text
   SECRET_KEY=ваш_секретный_ключ_django
   DEBUG=True
   ```
4. **Запустите локальный сервер Redis:**
   Убедитесь, что сервер Redis запущен на порту `6379`. При использовании Windows рекомендуется запускать со следующим флагом для избежания ошибок записи:
   ```bash
   redis-server.exe --stop-writes-on-bgsave-error no
   ```
5. **Примените миграции базы данных:**
   ```bash
   python manage.py migrate
   ```
6. **Создайте аккаунт администратора (Суперпользователя):**
   ```bash
   python manage.py createsuperuser
   ```
7. **Запустите сервер разработки Django:**
   ```bash
   python manage.py runserver
   ```
   Сайт будет доступен по адресу: `http://127.0.0`
